## Reflection

### 1. What went well during this unit?

Setting up the GitHub project board and milestones helped us organize our development workflow. We created issues for each feature and assigned them to milestones so the team knows exactly what to work on and when.

### 2. What were some challenges your group faced in this unit?

Getting comfortable with GitHub project management tools took some time. We also had to decide how to split development work across team members while avoiding merge conflicts.

### Did you finish all of your tasks in your sprint plan for this week? If you did not finish all of the planned tasks, how would you prioritize the remaining tasks on your list?

We completed the majority of our planned tasks including GitHub setup, project board creation, milestone assignments, and initial backend and frontend development. The remaining tasks are the two custom features (filter/sort and the claim modal) and deployment to Render. We would prioritize the custom features first since they are required for full credit, and then focus on deployment last since the app needs to be fully functional before going live.

### Which features and user stories would you consider “at risk”? How will you change your plan if those items remain “at risk”?

The dish claim modal is the most at risk because it involves a more complex UI interaction with a pop-up overlay and communication between multiple components. If it remains at risk, we would simplify the implementation by using a basic dropdown on the event page instead of a full modal, which would still allow users to claim dishes without navigating away. The filter and sort feature is lower risk since it mainly involves query parameters on the backend and state management on the frontend.

### 5. What additional support will you need in upcoming units as you continue to work on your final project?

We need guidance on deploying both the Express backend and React frontend to Render and connecting them in production.
